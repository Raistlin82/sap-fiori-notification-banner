================================================================================
  SAP TRANSPORT REQUEST - NOTIFICATION BANNER DEPLOYMENT
================================================================================

STEP 1: CREATE TRANSPORT REQUEST
---------------------------------
Transaction: SE10 or SE09

1. Go to SE10
2. Click "Create" button
3. Select "Customizing Request" or "Workbench Request" (use Workbench for dev objects)
4. Enter description: "Global Notification Banner - Initial Setup"
5. Note the Transport Request Number (e.g., DEVK900123)


STEP 2: CREATE DATABASE TABLE (ZTNOTIFY_MSGS)
----------------------------------------------
Transaction: SE11

1. SE11 → Database Table
2. Table name: ZTNOTIFY_MSGS
3. Click "Create"
4. Enter the fields as defined in file: ztnotify_msgs.se11
5. Delivery Class: A (Application table)
6. Click "Save" → Assign to your transport request
7. Click "Activate"


STEP 3: CREATE CDS VIEW (ZT_NOTIFY_MESSAGES)
---------------------------------------------
Transaction: SE80 or ADT (ABAP Development Tools)

Using SE80:
1. SE80 → Select "Repository Browser"
2. Create → Dictionary Objects → Data Definition
3. Name: ZT_NOTIFY_MESSAGES
4. Copy content from file: ztnotify_messages.ddls
5. Save → Assign to transport request
6. Activate

Using ADT (Eclipse):
1. Right-click on package → New → Other Repository Object → Core Data Services → Data Definition
2. Name: ZT_NOTIFY_MESSAGES
3. Paste content from ztnotify_messages.ddls
4. Save and activate


STEP 4: CREATE ABAP CLASSES
----------------------------
Transaction: SE24 or SE80

Class 1: ZCL_NOTIFICATION_MANAGER
1. SE24 → Create Class
2. Class name: ZCL_NOTIFICATION_MANAGER
3. Copy code from: zcl_notification_manager.clas.abap
4. Save → Assign to transport
5. Activate

Class 2: ZCL_NOTIFICATION_REST
1. SE24 → Create Class
2. Class name: ZCL_NOTIFICATION_REST
3. Superclass: CL_REST_RESOURCE
4. Copy code from: zcl_notification_rest.clas.abap
5. Save → Assign to transport
6. Activate


STEP 5: CONFIGURE REST SERVICE
-------------------------------
Transaction: SICF

1. Go to SICF
2. Navigate to: /default_host/sap/bc/rest/
3. Right-click → New Sub-Element
4. Service name: zcl_notification_rest
5. Description: Notification REST API
6. Handler List tab:
   - Handler Class: ZCL_NOTIFICATION_REST
7. Save and activate service
8. Test: http://your-server:port/sap/bc/rest/zcl_notification_rest/


STEP 6: CREATE AUTHORIZATION OBJECTS
-------------------------------------
See file: AUTHORIZATION_SETUP.txt for complete role configuration


STEP 7: RELEASE TRANSPORT
--------------------------
Transaction: SE10

1. Go to SE10
2. Find your transport request
3. Select all tasks
4. Release tasks (icon or F9)
5. Release main request
6. Export transport to target systems


STEP 8: IMPORT TO QA/PROD
--------------------------
Transaction: STMS (Transport Management System)

1. Go to STMS
2. Select import queue for target system
3. Find your transport
4. Import (with option "Import again" if needed)
5. Verify logs for errors


VERIFICATION CHECKLIST
----------------------
□ Table ZTNOTIFY_MSGS exists and activated
□ CDS View ZT_NOTIFY_MESSAGES activated
□ Class ZCL_NOTIFICATION_MANAGER activated
□ Class ZCL_NOTIFICATION_REST activated
□ REST service /sap/bc/rest/zcl_notification_rest/ active
□ Authorization objects created
□ Test data inserted via SM30 (optional)
□ REST API responds to GET requests
□ UI5 app can connect to backend


ROLLBACK PROCEDURE
------------------
If deployment fails:

1. SE10 → Find your transport
2. Utilities → Delete
3. Manually delete objects if needed:
   - SE11: Delete table ZTNOTIFY_MSGS
   - SE24: Delete classes ZCL_NOTIFICATION_*
   - SICF: Deactivate/delete REST service
4. PFCG: Delete authorization objects


TROUBLESHOOTING
---------------
Issue: "Object already exists"
→ Solution: Check if object exists in another package, use different name prefix

Issue: "Transport not found"
→ Solution: Verify transport number, check SE10 logs

Issue: "Activation failed"
→ Solution: Check syntax errors, dependencies, authorizations

Issue: "REST service returns 500"
→ Solution: ST22 for ABAP dumps, check class implementation
